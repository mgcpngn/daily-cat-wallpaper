# Daily Cat Wallpaper

[English](README.md) | [简体中文](README.zh-CN.md) | [繁體中文](README.zh-TW.md) | 日本語 | [한국어](README.ko.md)

Daily Cat Wallpaper は、Rust と Tauri 2 で作られたクロスプラットフォームのデスクトップアプリです。単なる猫の壁紙ローテーションではなく、ユーザーのフィードバックから学習する猫の壁紙体験を目指しています。高品質な猫画像を探す、または生成し、実際のモニター構成に合わせて壁紙を合成し、結果を分析し、ユーザーの評価を次回以降の選択に反映します。

Windows が最初の安定ターゲットです。macOS と Linux は静的壁紙の設定に対応していますが、デスクトップ環境ごとの差が大きいため、現時点では Beta として扱います。

## プロダクト方針

現在の目標は「Bing Wallpaper をランダムな猫画像に置き換えること」ではありません。より意図に合ったデスクトップ猫体験を作ることです。

- 猫は、ユーザーが選んだ品種、色、数、タイプ、雰囲気、シーンにできるだけ合うべきです。
- 2K 未満の画像は管理ギャラリーに入れません。
- 透明な猫画像は、猫が画面上にいるように見えるデスクトップペット風の壁紙に使えるべきです。
- マルチモニターでは、デフォルトでディスプレイ数に合わせて異なる猫を使います。ユーザーが 1 匹を明示的に選んだ場合は、複数画面で同じ猫を使えます。
- ユーザーは、ダウンロード済み画像や生成画像を閲覧し、好きな画像を手動で選び、気に入らない画像を削除できる必要があります。
- アプリは閉ループを持つ必要があります。実際のデスクトップ結果を分析し、好き/嫌いと理由を記録し、拒否された画像が再び選ばれる可能性を下げます。

## 主な機能

- Rust コア、Tauri 2 デスクトップシェル、React 設定 UI。
- UI と README は英語をデフォルトにし、簡体字中国語、繁体字中国語、日本語、韓国語に対応。
- Windows では `SystemParametersInfoW` によるネイティブな静的壁紙設定を行い、毎回仮想デスクトップ全体に合う壁紙を生成します。
- macOS は AppleScript による壁紙設定に対応し、現在は Beta です。
- Linux は GNOME、Cinnamon、Unity、KDE Plasma、Xfce にベストエフォートで対応し、現在は Beta です。
- TheCatAPI、Wikimedia Commons、ローカルフォルダー、Pexels、Pixabay、Magnific、AI 生成素材を対象に、品種と好みに基づいて候補を計画します。
- 強制 HD ポリシー: `2560x1440` 未満の画像は、管理ギャラリーへ保存する前に拒否されます。
- アプリデータディレクトリ内に固定の管理ギャラリーを持ち、ダウンロード画像と AI 生成画像を分けて保存します。
- ギャラリーブラウザーで、プレビュー、品質バッジ、手動適用、削除ができます。
- AI による透明な猫 PNG 生成に対応します。OpenAI は透明背景に対応した画像モデルをデフォルトで使い、Google Nano Banana Pro はユーザー API キーで使う Beta プロバイダーです。
- デスクトップ効果分析は、最終壁紙のサイズ、仮想デスクトップへの適合、解像度、プレースホルダー画像、透明素材、切り抜きの可能性、好みとの一致の不確実性を確認します。
- フィードバック学習は、好き/嫌いと理由を記録し、繰り返し拒否された画像を次回以降避けます。
- 呼吸、マウス接近、猫の手クリック、キーボード Bongo、サウンドなどのインタラクション設定に対応します。サウンドはデフォルトでオフです。

## 閉ループ学習

Daily Cat Wallpaper は壁紙選択を継続的に改善するシステムとして扱います。

1. 品種、色/タイプ、猫の数、シーン、ソース優先度、更新頻度、インタラクション、言語を読み取ります。
2. ローカルファイル、管理ギャラリー、オンラインソース、AI 生成から候補画像を解決します。
3. 低解像度画像は保存前に拒否します。
4. 実際の仮想デスクトップに合わせて壁紙を合成し、タスクバーとアイコン領域にも配慮します。
5. 最終結果を分析し、サイズ不足、プレースホルダー画像、透明でないペット素材、切り抜きの可能性、好みとの一致が未確認であることを報告します。
6. ユーザーが結果を好き/嫌いで評価し、理由を入力できます。
7. フィードバックをアプリデータディレクトリに保存し、拒否された画像を次回以降避けます。

この仕組みが、アプリをユーザーの好みに近づけていく中心です。

## 画像品質ポリシー

デフォルトの最小解像度は `2560x1440` です。低解像度フォールバックは無効です。ダウンロード画像はディスクに保存される前に検査されるため、弱い候補が管理ギャラリーを汚しません。

既存のローカルファイルは、ギャラリー内で品質バッジ付きで表示できます。ユーザーは手動で整理できます。

## AI 猫画像生成

AI 生成パネルは、現在の UI 設定からプロンプトを組み立てます。品種、タイプ/色、シーン、数量、透明背景の設定が反映されます。

生成画像の保存先:

- `cat-gallery/generated`

ダウンロード画像の保存先:

- `cat-gallery/downloads`

透明猫画像の目標は、背景なし、全身が見える、生き生きした PNG cutout です。これらの素材はデスクトップサイズの静的壁紙に合成されます。本当の透明オーバーレイ型デスクトップペット層は今後の作業です。現在のリリースは、静的壁紙合成とフィードバック学習に重点を置いています。

## 画像ソース

デフォルトおよび任意の画像ソース:

| ソース | 状態 | メモ |
| --- | --- | --- |
| ローカルフォルダー | Stable | ユーザーが選んだ HD 画像に最適です。 |
| 管理ギャラリー | Stable | ダウンロード先と生成先が固定されています。 |
| Wikimedia Commons | 公開フォールバック | 可能な範囲で品種とタイプのキーワードを使います。 |
| TheCatAPI | 公開/API キー対応 | 既知の品種は breed ID にマッピングされます。 |
| CATAAS | 公開フォールバック | 最後のオンラインフォールバックとして使えます。 |
| Pexels | 任意キー | API キーがある場合の高品質ソースです。 |
| Pixabay | 任意キー | API キーがある場合の高品質ソースです。 |
| Magnific | Experimental | API の利用可否とユーザー設定に依存します。 |
| OpenAI Images | 任意キー | 透明猫 cutout のデフォルト生成元です。 |
| Google Nano Banana Pro | 任意キー、Beta | 設定されたシーンに基づく画像生成プロバイダーです。 |

## 設定

設定 UI で扱う項目:

- 言語: 自動検出または明示的な選択。
- 猫の品種とタイプ/色の好み。
- 猫の数: デフォルトではディスプレイ数に合わせるか、固定で `1..5`。
- 画像の雰囲気とシーン。
- 画像ソースの優先度と API キー。
- ローカルギャラリーフォルダー。
- AI 生成プロバイダー、モデル、シーンプロンプト、数量、自動使用。
- 更新スケジュール: ログイン時、毎日指定時刻、一定間隔、手動更新。
- プラットフォームモードと自動起動。
- インタラクション設定とサウンド切り替え。

## インストール

最新ビルドは GitHub Releases からダウンロードできます。

[Daily Cat Wallpaper Releases](https://github.com/mgcpngn/daily-cat-wallpaper/releases)

公開対象は、GitHub Actions のビルドマトリクスが対応する Windows、macOS、Linux の x64 と arm64 です。

## 開発

前提条件:

- `rustfmt` と `clippy` を含む Rust stable toolchain。
- Node.js 22 以降。
- 使用 OS 向けの Tauri プラットフォーム依存関係。

依存関係をインストール:

```powershell
npm install
```

チェックを実行:

```powershell
cargo test --workspace
npm run build
cargo check
```

開発モードで実行:

```powershell
npm run tauri dev
```

本番ビルド:

```powershell
npm run tauri:build
```

## 現在の制限

- Windows が主要な安定ターゲットです。macOS と Linux の壁紙設定は Beta です。
- 透明 AI 猫画像は現在、静的壁紙に合成されます。本当の透明オーバーレイ型デスクトップペット層は未完成です。
- オンライン画像の品質は、プロバイダー API、レート制限、ユーザー API キーに依存します。
- 品種マッチングはベストエフォートです。ユーザーが結果を拒否した場合は、フィードバックループが次回以降の優先判断になります。
