# Daily Cat Wallpaper

[English](README.md) | [简体中文](README.zh-CN.md) | 繁體中文 | [日本語](README.ja.md) | [한국어](README.ko.md)

Daily Cat Wallpaper 是一個基於 Rust 和 Tauri 2 的跨平台桌面應用，用來實現帶回饋學習的「吸貓桌布自由」。專案已經不再只是簡單地定時換一張貓圖，而是一個可管理的桌面貓咪陪伴系統：它會尋找或生成高品質貓圖素材，依照真實顯示器布局合成桌布，分析最後桌面效果，並根據使用者回饋持續調整後續選擇。

Windows 是首個穩定目標平台。macOS 和 Linux 已支援基礎靜態桌布設定，但由於不同桌面環境的桌布介面差異較大，目前以 Beta 能力發布。

## 產品方向

目前目標不是「用隨機貓圖取代必應桌布」，而是讓桌面貓咪體驗更符合使用者預期：

- 貓咪應盡量匹配使用者選擇的品種、顏色、數量、類型、情緒和場景。
- 低於 2K 的圖片不應進入託管圖庫。
- 透明貓圖應支援類似桌寵的桌布效果，讓貓像真的待在螢幕上一樣，而不是被粗糙貼到背景圖裡。
- 多螢幕預設更合理：預設使用者有幾個螢幕就使用幾隻不同的貓；如果使用者明確選擇 1 隻貓，則可在多個螢幕上重複使用同一隻貓。
- 使用者必須能瀏覽已下載或生成的貓圖，手動選擇喜歡的圖片，並刪除不滿意的圖片。
- 系統必須形成閉環：分析實際生成的桌面效果，記錄喜歡/不喜歡和原因，並降低被拒絕圖片再次出現的機率。

## 功能亮點

- Rust 核心、Tauri 2 桌面殼、React 設定介面。
- UI 和 README 支援多語言，預設英文，並支援簡體中文、繁體中文、日文、韓文。
- Windows 使用 `SystemParametersInfoW` 原生接管靜態桌布，每次刷新都會生成適配虛擬桌面尺寸的桌布。
- macOS 透過 AppleScript 設定桌布，目前為 Beta。
- Linux 對 GNOME、Cinnamon、Unity、KDE Plasma、Xfce 做最佳努力支援，目前為 Beta。
- 面向品種和偏好的圖片源規劃，支援 TheCatAPI、Wikimedia Commons、本機目錄、Pexels、Pixabay、Magnific 和 AI 生成素材。
- 強制高清策略：低於 `2560x1440` 的圖片在寫入託管圖庫前會被拒絕。
- 固定託管圖庫位於應用資料目錄，並區分下載圖片和 AI 生成圖片。
- 內建圖庫瀏覽，支援預覽、品質標記、手動使用、刪除。
- 支援 AI 生成透明貓咪 PNG，用於桌寵風格桌布。OpenAI 預設使用支援透明背景的圖像模型；Google Nano Banana Pro 作為使用者自填 Key 的 Beta 提供方。
- 桌面效果體檢會檢查最後桌布尺寸、虛擬桌面鋪滿情況、解析度、占位圖、透明素材、可能裁切和偏好匹配不確定性。
- 回饋學習會記錄喜歡/不喜歡和原因，並在後續選擇中避開反覆被拒絕的圖片。
- 支援呼吸、滑鼠靠近、點擊貓掌、鍵盤 Bongo、聲音等互動偏好設定，聲音預設關閉。

## 閉環學習機制

Daily Cat Wallpaper 把桌布選擇當作一個持續迭代的系統：

1. 讀取使用者偏好：品種、顏色/類型、貓咪數量、場景、圖片源優先級、刷新頻率、互動形式和語言。
2. 從本機檔案、託管圖庫、線上來源或 AI 生成中解析候選圖片。
3. 在儲存前拒絕低解析度圖片。
4. 根據真實虛擬桌面合成桌布，並盡量考慮工作列和圖示安全區。
5. 分析最後桌布結果，提示尺寸不足、占位圖、非透明桌寵素材、可能裁切、偏好匹配不確定等問題。
6. 讓使用者對結果按喜歡或不喜歡，並填寫原因。
7. 在應用資料目錄持久化回饋，後續選擇時避開被拒絕的圖片。

這是系統越來越懂使用者的核心機制。

## 圖片品質策略

預設最低接受解析度是 `2560x1440`。低解析度兜底已停用。下載圖片會先通過解析度檢查，只有合格圖片才會寫入磁碟，避免託管圖庫被低品質候選污染。

已經存在的本機檔案仍可在圖庫中顯示品質標記，方便使用者手動清理。

## AI 貓圖生成

AI 生成面板會根據目前 UI 選項自動組織提示詞，包括品種、類型/顏色、場景、數量和透明背景偏好。

生成圖片儲存到：

- `cat-gallery/generated`

下載圖片儲存到：

- `cat-gallery/downloads`

透明貓圖的目標是栩栩如生、全身完整、無背景的 PNG cutout。這些素材會被合成到桌面尺寸的靜態桌布中。真正的透明懸浮桌寵互動層仍屬於後續工作；目前版本重點是靜態桌布合成和回饋學習閉環。

## 圖片源

預設和可選圖片源：

| 圖片源 | 狀態 | 說明 |
| --- | --- | --- |
| 本機目錄 | 穩定 | 最適合使用者自己精選高清圖片。 |
| 託管圖庫 | 穩定 | 固定下載和生成目錄。 |
| Wikimedia Commons | 公共兜底 | 盡量使用品種和類型關鍵字。 |
| TheCatAPI | 公共/API Key 友好 | 已知品種會映射到 breed ID。 |
| CATAAS | 公共兜底 | 作為最後的線上兜底來源。 |
| Pexels | 可選 Key | 使用者提供 API Key 後可作為高品質來源。 |
| Pixabay | 可選 Key | 使用者提供 API Key 後可作為高品質來源。 |
| Magnific | 實驗性 | 取決於 API 可用性和使用者設定。 |
| OpenAI Images | 可選 Key | 預設用於透明貓咪 cutout。 |
| Google Nano Banana Pro | 可選 Key，Beta | 根據設定場景生成圖像。 |

## 設定項

設定介面覆蓋：

- 語言：自動偵測系統語言或手動選擇。
- 貓咪品種和類型/顏色偏好。
- 貓咪數量策略：預設匹配顯示器數量，或固定為 `1..5`。
- 圖片情緒和場景。
- 圖片源優先級和 API Key。
- 本機圖庫目錄。
- AI 生成提供方、模型、場景提示、數量和自動使用策略。
- 刷新計畫：登入時、每天固定時間、間隔刷新或手動刷新。
- 平台模式和開機啟動。
- 互動偏好和聲音開關。

## 開發

前置條件：

- Rust stable toolchain，並包含 `rustfmt` 和 `clippy`。
- Node.js 22 或更新版本。
- 目前作業系統對應的 Tauri 平台依賴。

安裝依賴：

```powershell
npm install
```

執行檢查：

```powershell
cargo test --workspace
npm run build
cargo check
```

開發模式執行：

```powershell
npm run tauri dev
```

建置正式版本：

```powershell
npm run tauri:build
```

## 目前限制

- Windows 是目前主要穩定目標。macOS 和 Linux 桌布設定仍為 Beta。
- 透明 AI 貓圖目前會被合成為靜態桌布，真正的透明懸浮桌寵層尚未完成。
- 線上圖片品質受提供方 API、速率限制和使用者 API Key 影響。
- 品種匹配是最佳努力；當使用者拒絕結果時，回饋閉環才是後續選擇的優先依據。
