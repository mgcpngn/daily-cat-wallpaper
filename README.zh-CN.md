# Daily Cat Wallpaper

[English](README.md) | 简体中文 | [繁體中文](README.zh-TW.md) | [日本語](README.ja.md) | [한국어](README.ko.md)

Daily Cat Wallpaper 是一个基于 Rust 和 Tauri 2 的跨平台桌面应用，用来实现带反馈学习的「吸猫壁纸自由」。项目已经不再只是简单地定时换一张猫图，而是一个可管理的桌面猫咪陪伴系统：它会寻找或生成高质量猫图资产，按真实显示器布局合成壁纸，分析最终桌面效果，并根据用户反馈不断调整后续选择。

Windows 是首个稳定目标平台。macOS 和 Linux 已支持基础静态壁纸设置，但由于不同桌面环境的壁纸接口差异较大，当前按 Beta 能力发布。

## 产品方向

当前目标不是「用随机猫图替换必应壁纸」，而是让桌面猫咪体验更符合用户预期：

- 猫咪应尽量匹配用户选择的品种、颜色、数量、类型、情绪和场景。
- 低于 2K 的图片不应进入托管图库。
- 透明猫图应支持类似桌宠的壁纸效果，让猫像真的待在屏幕上一样，而不是被粗糙贴到背景图里。
- 多屏默认更合理：默认用户有几块屏幕就使用几只不同的猫；如果用户明确选择 1 只猫，则可在多块屏幕上复用同一只猫。
- 用户必须能浏览已经下载或生成的猫图，手动选择喜欢的图片，并删除不满意的图片。
- 系统必须形成闭环：分析实际生成的桌面效果，记录喜欢/不喜欢和原因，并降低被拒绝图片再次出现的概率。

## 功能亮点

- Rust 核心、Tauri 2 桌面壳、React 配置界面。
- UI 和 README 支持多语言，默认英语，并支持简体中文、繁体中文、日文、韩文。
- Windows 使用 `SystemParametersInfoW` 原生接管静态壁纸，每次刷新都会生成适配虚拟桌面尺寸的壁纸。
- macOS 通过 AppleScript 设置壁纸，当前为 Beta。
- Linux 对 GNOME、Cinnamon、Unity、KDE Plasma、Xfce 做最佳努力支持，当前为 Beta。
- 面向品种和偏好的图片源规划，支持 TheCatAPI、Wikimedia Commons、本地目录、Pexels、Pixabay、Magnific 和 AI 生成资产。
- 强制高清策略：低于 `2560x1440` 的图片在写入托管图库前会被拒绝。
- 固定托管图库位于应用数据目录，并区分下载图片和 AI 生成图片。
- 内置图库浏览，支持预览、质量标记、手动使用、删除。
- 支持 AI 生成透明猫咪 PNG，用于桌宠风格壁纸。OpenAI 默认使用支持透明背景的图像模型；Google Nano Banana Pro 作为用户自填 Key 的 Beta 提供方。
- 桌面效果体检会检查最终壁纸尺寸、虚拟桌面铺满情况、分辨率、占位图、透明资产、可能裁剪和偏好匹配不确定性。
- 反馈学习会记录喜欢/不喜欢和原因，并在后续选择中避开反复被拒绝的图片。
- 支持呼吸、鼠标靠近、点击猫爪、键盘 Bongo、声音等交互偏好配置，声音默认关闭。

## 闭环学习机制

Daily Cat Wallpaper 把壁纸选择当作一个持续迭代的系统：

1. 读取用户偏好：品种、颜色/类型、猫咪数量、场景、图片源优先级、刷新频次、交互形式和语言。
2. 从本地文件、托管图库、在线源或 AI 生成中解析候选图片。
3. 在保存前拒绝低分辨率图片。
4. 根据真实虚拟桌面合成壁纸，并尽量考虑任务栏和图标安全区。
5. 分析最终壁纸结果，提示尺寸不足、占位图、非透明桌宠资产、可能裁剪、偏好匹配不确定等问题。
6. 让用户对结果点赞或点踩，并填写原因。
7. 在应用数据目录持久化反馈，后续选择时避开被拒绝的图片。

这是系统越来越懂用户的核心机制。

## 图片质量策略

默认最低接受分辨率是 `2560x1440`。低分辨率兜底已禁用。下载图片会先通过分辨率检查，只有合格图片才会写入磁盘，避免托管图库被低质量候选污染。

已经存在的本地文件仍可在图库中显示质量标记，方便用户手动清理。

## AI 猫图生成

AI 生成面板会根据当前 UI 选项自动组织提示词，包括品种、类型/颜色、场景、数量和透明背景偏好。

生成图片保存到：

- `cat-gallery/generated`

下载图片保存到：

- `cat-gallery/downloads`

透明猫图的目标是活灵活现、全身完整、无背景的 PNG cutout。这些资产会被合成到桌面尺寸的静态壁纸中。真正的透明悬浮桌宠交互层仍属于后续工作；当前版本重点是静态壁纸合成和反馈学习闭环。

## 图片源

默认和可选图片源：

| 图片源 | 状态 | 说明 |
| --- | --- | --- |
| 本地目录 | 稳定 | 最适合用户自己精选高清图片。 |
| 托管图库 | 稳定 | 固定下载和生成目录。 |
| Wikimedia Commons | 公共兜底 | 尽量使用品种和类型关键词。 |
| TheCatAPI | 公共/API Key 友好 | 已知品种会映射到 breed ID。 |
| CATAAS | 公共兜底 | 作为最后的在线兜底源。 |
| Pexels | 可选 Key | 用户提供 API Key 后可作为高质量来源。 |
| Pixabay | 可选 Key | 用户提供 API Key 后可作为高质量来源。 |
| Magnific | 实验性 | 取决于 API 可用性和用户配置。 |
| OpenAI Images | 可选 Key | 默认用于透明猫咪 cutout。 |
| Google Nano Banana Pro | 可选 Key，Beta | 根据配置场景生成图像。 |

## 配置项

配置界面覆盖：

- 语言：自动检测系统语言或手动选择。
- 猫咪品种和类型/颜色偏好。
- 猫咪数量策略：默认匹配显示器数量，或固定为 `1..5`。
- 图片情绪和场景。
- 图片源优先级和 API Key。
- 本地图库目录。
- AI 生成提供方、模型、场景提示、数量和自动使用策略。
- 刷新计划：登录时、每天固定时间、间隔刷新或手动刷新。
- 平台模式和开机启动。
- 交互偏好和声音开关。

## 开发

前置条件：

- Rust stable toolchain，并包含 `rustfmt` 和 `clippy`。
- Node.js 22 或更新版本。
- 当前操作系统对应的 Tauri 平台依赖。

安装依赖：

```powershell
npm install
```

运行检查：

```powershell
cargo test --workspace
npm run build
cargo check
```

开发模式运行：

```powershell
npm run tauri dev
```

构建生产版本：

```powershell
npm run tauri:build
```

## 当前限制

- Windows 是当前主要稳定目标。macOS 和 Linux 壁纸设置仍为 Beta。
- 透明 AI 猫图目前会被合成为静态壁纸，真正的透明悬浮桌宠层尚未完成。
- 在线图片质量受提供方 API、速率限制和用户 API Key 影响。
- 品种匹配是最佳努力；当用户拒绝结果时，反馈闭环才是后续选择的优先依据。
